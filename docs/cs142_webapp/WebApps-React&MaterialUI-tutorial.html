01 Core Layout

Grid Layout with Material-UI

The foundation of any Material-UI app is the responsive Grid system. It uses a 12-column layout that adapts to different screen sizes. Use a container Grid wrapping item Grids to create your app shell.

xs=12 — TopBar (full width)
AppBar / TopBar
xs=4
UserList
xs=8
UserDetail or UserPhotos 
<Grid container spacing={8}>

  {/* Top bar — full width at all sizes */}
  <Grid item xs={12}>
    <TopBar />
  </Grid>

  {/* Sidebar — 4 columns */}
  <Grid item xs={4}>
    <UserList />
  </Grid>

  {/* Content — 8 columns */}
  <Grid item xs={8}>
    <UserDetail />
    {/* or <UserPhotos /> */}
  </Grid>

</Grid>

03 Deep Linking

React Router for Deep Linking

To support bookmarking and URL sharing, use React Router. The content area becomes a <Switch> that renders the right component based on the current URL path.

Why deep linking? Without routing, refreshing the page always returns users to the home screen. Deep linking preserves the current view state in the URL. 
// Route configuration in the content area
<Switch>
  <Route path="/users/:userId"   component={UserDetail} />
  <Route path="/photos/:userId"  component={UserPhotos} />
  <Route path="/users"          component={UserList} />
</Switch>

// Navigation links in the sidebar
<Link to="/photos/57231f1a30e4351f4e9f4bd8">
  Photos of User Ellen Ripley
</Link>

04 Responsive Design

Material UI Breakpoints & Responsive Support

Material-UI uses CSS flexbox for relative sizing, with named breakpoints to handle significant layout differences at specific screen widths.

xs
0px+
sm
600px+
md
960px+
lg
1280px+
xl
1920px+

Apply breakpoints in three ways:

// 1. Grid: column widths per breakpoint
<Grid item xs={12} md={6} xl={3}> ...

// 2. Hidden: conditional rendering
<Hidden mdUp>
  <Paper>Only visible below md</Paper>
</Hidden>

// 3. useMediaQuery hook
const theme = useTheme();
const matches = useMediaQuery(theme.breakpoints.up('md'));
if (matches) { /* large screen logic */ }
05 Accessibility

Accessible Rich Internet Applications (ARIA)

Accessibility ensures your app works for all users, including those using assistive technologies. ARIA attributes bridge the gap between visual interfaces and screen readers.

<!-- Text alternative for a link -->
<a aria-label="Photo of user Ellen Ripley" href="...">

<!-- Text alternative for an image -->
<img aria-label="A photo of a sunset" src="..." />
🔍

Zoom Support

Work correctly when zoomed in; avoid quick timeouts

🎨

High Contrast

Sufficient contrast between foreground and background

⌨️

Keyboard Nav

Full functionality without a mouse

📝

Simple Language

Clear, concise sentences throughout

06 Internationalization

Internationalization (I18N)

Users around the world want different text, date formats, number formats, currencies, and graphics. I18N requires a level of indirection — never hardcode user-facing strings.

Skip user-generated content: Don't run user-generated content (e.g. photo captions, comments) through the translation system. 
// ❌ Hardcoded — not internationalizable
<h1>Getting Started</h1>

// ✅ With react-i18next — key-based lookup
<Trans i18nKey="userMessagesUnread" count={count}>
  Hello <strong title={t('nameTitle')}>{{name}}</strong>,
  you have {{count}} unread message.
</Trans>
07 Testing

Testing Web Applications

A solid testing strategy combines unit tests for individual components with end-to-end tests that exercise the full application as a user would.

TypeScopeApproachTooling
Unit Tests Single component Mock all dependencies; test in isolation Jest, React Testing Library
E2E Tests Full application Drive real browser programmatically Selenium, Playwright (WebDriver)
Test Coverage: Aim for every line of code to have at least one test. Coverage tools like Istanbul/NYC report which lines are untested.

08 Design Goals

Good Web App Design Principles

The hardest part of good web applications is the design. Technical excellence alone is not enough — good user interface principles must be followed.

🧠

Intuitive

No manual required. Users discover functionality naturally.

⚡

Accurate & Fast

Provides needed info; users accomplish tasks quickly.

😊

Enjoyable

Joy, not pain. The experience itself matters.

Three guiding principles: Be consistent (reduce cognitive load), provide context (user shouldn't get lost), be fast (don't make users wait).
09 Consistency

Style Guides & Design Templates

Web apps need a style guide to define their look and feel consistently across every screen:

  • Style — Color schemes, animation, icons, images, typography, writing
  • User interactions — Menus, buttons, pickers, dialog boxes, tables, lists
  • Layout — Structure, toolbars, content, responsiveness
  • Patterns — Do the same thing the same way, everywhere (error handling, navigation, notifications)
  • Design templates — Follow familiar structures like the master-detail pattern
10 Material Design

Material Design from Google

Material Design is Google's design system used across Android and web apps. It's inspired by publishing (paper and ink) and enhanced with technology — heavy use of animation to convey action.

FoundationDescription
EnvironmentSurfaces (like paper), depth, and shadows
LayoutResponsive layout grid, breakpoints, white space
NavigationLateral, Forward, and Backward view changes
ColorHarmonious color recommendations
TypographyPoint size, weight, and spacing guidelines
IconographyLanguage-independent visual expressions
ShapeDirect attention, identify, communicate
MotionShow relationships, focus attention
InteractionMap touch gestures to actions
CommunicationWriting, formats, imagery, onboarding
11 Frameworks

Front-End Web Frameworks

🅱️

Bootstrap

CSS design templates, grid system, element styling, HTML components (buttons, menus, forms), and JS utilities (modals, dropdowns). Originally jQuery-based.

⚛️

Material-UI

The popular UI library for React. Implements the full Material Design spec with ready-made components and a theming system.

Note: ReactJS itself has no opinion on UI. Material-UI (now called MUI) is the popular community choice for Material Design in React.
