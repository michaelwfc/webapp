01

What is Express.js?

Express.js is a thin layer on top of Node.js that makes building web servers significantly easier. It's described as fast, unopinionated, and minimalist — meaning it gives you tools without forcing you into rigid patterns.

🌐

HTTP Handling

Accept TCP connections, process requests, and send replies. Built on Node's HTTP module.

🗺️

Routing

Map URLs to handler functions. Supports a routing table similar to React Router.

🔌

Middleware

Plug in sessions, cookies, security, compression, and custom logic with ease.

02

Basic Setup

Express uses the factory pattern — you call express() to get your app object, then use it to register routes and start the server.

JavaScript 
let express = require('express');
let expressApp = express(); // module uses factory pattern

// expressApp has methods for:
//   Routing HTTP requests
//   Rendering HTML (e.g. Jade templating engine)
//   Configuring middleware and preprocessors

expressApp.get('/', function (httpRequest, httpResponse) {
  httpResponse.send('hello world');
});

expressApp.listen(3000); // localhost:3000
💡
Calling expressApp.listen(3000) starts the server on localhost:3000 by default. You can pass a callback as the second argument to log a message when the server starts.
03

Express Routing

Express maps HTTP methods and URL paths to handler functions. Each method corresponds directly to an HTTP verb.

JavaScript 
expressApp.get(urlPath, requestProcessFunction);     // GET
expressApp.post(urlPath, requestProcessFunction);    // POST
expressApp.put(urlPath, requestProcessFunction);     // PUT
expressApp.delete(urlPath, requestProcessFunction);  // DELETE
expressApp.all(urlPath, requestProcessFunction);     // ALL methods

Route Parameters

Like React Router, Express supports dynamic URL segments using the : prefix. These become available in request.params.

JavaScript 
// URL: /user/42  →  request.params.user_id === "42"
expressApp.get('/user/:user_id', function (req, res) {
  let id = req.params.user_id;
  res.send('User ID is: ' + id);
});
04

The Request Object

The httpRequest object carries all information about the incoming request. Middleware can add extra properties to it.

Property / MethodDescriptionExample
request.paramsURL route parameters{ user_id: '42' }
request.queryQuery string parameters&foo=9 → { foo: '9' }
request.bodyParsed request body (set by middleware){ name: 'Alice' }
request.get(field)Returns the value of an HTTP headerrequest.get('Content-Type')
ℹ️
request.body is only populated when body-parsing middleware is applied (e.g. express.json() or a JSON body parser). Without middleware, it will be undefined.
05

The Response Object

The httpResponse object is used to build and send the HTTP reply. Methods return this, enabling method chaining.

MethodDescription
response.write(content)Append content to the response body
response.status(code)Set the HTTP status code (200, 400, 500…)
response.set(prop, value)Set a response header property
response.end()Finalize and send the response
response.end(msg)Finalize and send with a message
response.send(content)Shorthand for write() + end()
response.json(obj)Send JSON (sets Content-Type automatically)

Method Chaining

Because methods return this, you can chain them for concise response handling:

JavaScript 
// Set status, write multiple chunks, then end
response.status(200).write(content1).write(content2).end();

// Most common: status + send in one line
response.status(200).send(userData);
⚠️
Always call end() or send() to close the response. Forgetting to finalize a response will cause the request to hang indefinitely.

Route-based Middleware

Apply middleware to a specific URL path using expressApp.all():

JavaScript 
expressApp.all(urlPath, function (request, response, next) {
  // Do processing on request or set up response
  next(); // pass control to the next handler
});

Global Middleware

Apply middleware to all requests using expressApp.use():

JavaScript 
expressApp.use(function (request, response, next) {
  // Runs for every incoming request
  next();
});

// Serve static files from the current directory
app.use(express.static(__dirname));

Complete Examples

Simple File Server (Project #4 Pattern)

Serve static files from a directory and handle a root route:

JavaScript — webServer.js 
let express = require('express');
let app = express();

app.use(express.static(__dirname)); // Serve files from this directory

app.get('/', function (request, response) {
  response.send('Simple web server of files from ' + __dirname);
});

app.listen(3000, function () {
  console.log('Listening at http://localhost:3000');
});

REST API Routes (Project #5 Pattern)

Fetch a list of users or a single user by ID, with proper error handling:

JavaScript 
// GET /user/list — return all users
app.get('/user/list', function (request, response) {
  response.status(200).send(cs142models.userListModel());
});

// GET /user/:id — return a single user
app.get('/user/:id', function (request, response) {
  let id = request.params.id;
  let user = cs142models.userModel(id);

  if (user === null) {
    response.status(400).send('Not found');
    return;
  }
  response.status(200).send(user);
});

Reading from a JSON File

Fetch an object from the filesystem and return it as JSON:

JavaScript 
expressApp.get("/object/:objid", function (request, response) {
  let dbFile = "DB" + request.params.objid;

  fs.readFile(dbFile, function (error, contents) {
    if (error) {
      response.status(500).send(error.message);
    } else {
      let obj = JSON.parse(contents); // Buffer types accepted
      obj.date = new Date();
      response.json(obj); // sets Content-Type automatically
    }
  });
});
08

Async Patterns

When a route needs to fetch multiple related files or resources, you'll need an async coordination strategy. The async library's each function is a classic approach:

JavaScript — Fetching multiple models 
app.get("/commentsOf/:objid", function (request, response) {
  let comments = [];

  // Step 1: Read the main object file
  fs.readFile("DB" + request.params.objid, function (error, contents) {
    let obj = JSON.parse(contents);
    // Step 2: Fetch each comment file in parallel
    async.each(obj.comments, fetchComments, allDone);
  });

  // Called for each comment file
  function fetchComments(commentFile, callback) {
    fs.readFile("DB" + commentFile, function (error, contents) {
      if (!error) comments.push(JSON.parse(contents));
      callback(error);
    });
  }

  // Called when all fetches complete (or one errors)
  function allDone(error) {
    if (error) response.status(500).send(error.message);
    else response.json(comments);
  }
});
ℹ️
async.each runs all iterations in parallel and calls allDone once every callback has fired — or immediately if any callback receives an error. This is the classic pattern before Promise.all() became mainstream.

Modern Alternative with Promises

Today you'd typically use async/await and Promise.all() for cleaner async handling:

JavaScript — Modern async/await 
app.get("/commentsOf/:objid", async function (req, res) {
  try {
    const contents = await fs.promises.readFile("DB" + req.params.objid);
    const obj = JSON.parse(contents);
    const comments = await Promise.all(
      obj.comments.map(f => fs.promises.readFile("DB" + f).then(JSON.parse))
    );
    res.json(comments);
  } catch (err) {
    res.status(500).send(err.message);
  }
});
