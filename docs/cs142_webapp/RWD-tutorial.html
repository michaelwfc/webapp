Section 02

What is Responsive Web Design?

The core insight of RWD, coined by Ethan Marcotte, is elegant:

💧 Content is like water

Your web app should flow into and fill whatever device you have — just like water takes the shape of its container.

RWD is made possible with three CSS-level techniques:

01 / LAYOUT

Fluid Grid

Use relative units like % instead of absolute px. Elements pack into columns and rows, proportionally.

02 / QUERIES

Media Queries

CSS @media rules let you switch layouts, hide elements, or change functionality based on screen size.

03 / MEDIA

Flexible Media

Images and videos auto-scale to fit their container, never overflowing or distorting.

Flexible Images & Videos

Two CSS lines are all you need to make media responsive:

img   { width: 100%; height: auto; }
video { width: 100%; height: auto; }
✓ Why height: auto?

Setting height: auto preserves the original aspect ratio as the width changes. Without it, images would squish or stretch.

Section 03

Responsive Layout in Action

A classic RWD layout uses percentage-based columns. On wide screens menus sit side by side; on narrow screens they stack. Resize the demo below:

Preview:

How the Layout Works

The layout uses a single HTML structure. CSS @media rules change the widths:

/* Wide: nav sidebar (25%) + content (75%) */
@media (min-width: 768px) {
  .nav     { width: 25%; float: left; }
  .content { width: 75%; float: left; }
  .menu-item { width: 25%; }
}

/* Narrow: everything stacks to 100% */
@media (max-width: 767px) {
  .nav, .content { width: 100%; float: none; }
  .menu-item      { width: 100%; }
}
Section 04

CSS Breakpoints

Breakpoints are the specific widths where your layout "breaks" and needs to reorganize. They're defined using CSS @media rules.

767px
1024px
Mobile (≤767px)
Tablet (768–1024px)
Desktop (>1024px)

Common Breakpoint Patterns

/* Tablets and desktops */
@media only screen and (min-width: 768px) {
  /* Multi-column layout */
}

/* Phones */
@media only screen and (max-width: 767px) {
  /* Single column, stacked menus */
}

/* Portrait phones specifically */
@media only screen and (max-width: 767px)
  and (orientation: portrait) {
  /* Portrait-specific overrides */
}
✓ Orientation Queries

The orientation: portrait and orientation: landscape media features let you tailor layouts for how the user is holding their device — very useful for phones.

Phone Layout (max-width: 767px)

When the screen narrows, nav items and menus that were side-by-side each become full width, stacking vertically:

/* Each menu takes full row on mobile */
@media (max-width: 767px) {
  .menu-item { width: 100%; display: block; }
  .view      { width: 100%; }
  .footer    { width: 100%; }
}
Section 05

Responsive Implementation

Core Rules

Three foundational practices underpin every responsive implementation:

1

Relative over Absolute

Use %, em, rem, vw, vh instead of px or pt for layout dimensions. This lets elements scale naturally with the viewport.

2

CSS Breakpoints for Layout & Functionality

Use media queries not just to rearrange layout, but also to conditionally show or hide features based on available screen real estate. A complex data table might become a simplified card list on mobile.

3

Mobile First

Start with a great mobile experience and progressively enhance it for larger screens. This ensures the core content is always accessible, and you add complexity only when you have space for it.

Mobile First in CSS

In a mobile-first stylesheet, base styles are for small screens. You then override with min-width queries as the screen gets bigger:

/* BASE: mobile styles (no media query needed) */
.container {
  width: 100%;
  padding: 1rem;
}

/* TABLET: expand layout from 768px up */
@media (min-width: 768px) {
  .container { max-width: 960px; margin: 0 auto; }
  .sidebar   { width: 25%; float: left; }
  .main      { width: 75%; float: left; }
}

/* DESKTOP: full layout from 1200px up */
@media (min-width: 1200px) {
  .container { max-width: 1400px; }
}
📐 Device Independent Units

High-DPI (Retina) screens have more physical pixels per CSS pixel. Always specify sizes in CSS pixels (device-independent units) — the browser handles the mapping to physical pixels automatically.

Quick Reference

/* ✅ Responsive image */
img { width: 100%; height: auto; }

/* ✅ Fluid typography */
h1  { font-size: clamp(1.8rem, 5vw, 4rem); }

/* ✅ Flexible grid */
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
}

/* ✅ Viewport meta tag (required in HTML!) */
<meta name="viewport" content="width=device-width, initial-scale=1.0">
